# Project Overview

This is a **React 19 web admin dashboard** built with **Vite**, used to manage the Rei do Bar backend (products, categories, customers, orders). It is a single-page application with **file-based routing** via TanStack Router, a dark-themed UI, and TypeScript. The React Compiler is enabled, so manual memoization is generally unnecessary.

---

## Technology Stack

### Core
- **React 19** — UI library; the **React Compiler** (`babel-plugin-react-compiler`) is enabled through the Vite React plugin, so `useMemo`/`useCallback`/`memo` are rarely needed
- **Vite** — dev server and production build (`tsc -b && vite build`)
- **TypeScript** — strict null checks, no unused locals/parameters, and `verbatimModuleSyntax` (type-only imports **must** use `import type`)

### Routing
- **@tanstack/react-router** — file-based routing derived from `src/routes/`; the router Vite plugin regenerates `routeTree.gen.ts` (with automatic code splitting). Router devtools are mounted in the root route

### Data Fetching
- **@tanstack/react-query** (v5) — server state; `useQuery` and `useMutation` are the primary primitives
- **axios** — HTTP client; a shared, typed `api` instance in the services layer is used by all service hooks
- **react-hook-form** — form state; `register` for native inputs, `Controller` for custom controls
- **yup** + **@hookform/resolvers** — schema validation via `yupResolver`

### Tables
- **@tanstack/react-table** — headless table model; consumed by the shared data-table component

### UI Primitives
- **@radix-ui/react-\*** — accessible primitives (dialog, select, switch, tooltip) wrapped by shared components
- **motion** (`motion/react`) — animations (`motion.*`, `AnimatePresence`)
- **react-icons** — icon component sets (`md`, `fi`, `bi`, `hi`, `io5`, `lu`, `fa`, `pi`)
- **sonner** — toast notifications; a single `<Toaster>` is mounted at the app root and triggered via `toast()`

### Styling
- **tailwindcss v4** — utility-first styling applied through `className`, enabled by the `@tailwindcss/vite` plugin and configured CSS-first in `index.css` (there is **no** `tailwind.config.js`). The UI uses the **default Tailwind palette on a dark theme** — there are **no custom design tokens**
- **tailwind-merge** (`twMerge`) — merges conflicting utility classes; used by components that accept an external `className`

### State
- Client state lives in **route components** (local `useState`) and in **URL search params**. **zustand** is present in dependencies but is **not currently used** in `src/`

### Tooling
- **Biome** — single tool for linting + formatting (2-space indent, double quotes); configured in `biome.json`, which ignores the generated `routeTree.gen.ts`
- **Husky** — Git hooks (`prepare` script)

## Folder Structure

```
src/
  main.tsx           App entry: Router + QueryClientProvider + Toaster
  index.css          Tailwind import and global styles
  routeTree.gen.ts   Generated route tree — never edit by hand
  routes/            File-based routes and their feature-scoped code (TanStack Router)
  components/        Shared, reusable UI components (barrel: index.ts)
    form/            Form controls (input, select, button, toggle, ...)
    modal/           Modal primitives (Radix Dialog wrappers)
    page/            Page-level state components (wrapper, loading, error)
    table/           Headless data-table compound component + partials/
  shared/
    services/        Data-fetching hooks (axios + React Query) — one folder per resource (barrel)
    models/          API entity interfaces and enums (barrel)
    interfaces/      Cross-cutting structural API types: pagination, sort (barrel)
    helpers/         Pure utility functions (no barrel — imported by full path)
```

`public/` holds static assets. `routeTree.gen.ts` is regenerated by the router plugin and must not be edited manually.

## Global Conventions

### File & Directory Naming
- **kebab-case** for files and directories
- Route files follow TanStack Router conventions: `(group)` route groups (no URL segment), `$param` dynamic segments, and a leading `-` to exclude a file or folder from routing (`-partials/`, `-helpers/`, `-shared/`, `-types.ts`)

### Exports
- **Named exports everywhere** — there are no default exports (route files export a named `Route`)
- Barrel `index.ts` files aggregate exports for `components/`, each `-partials/` and `-helpers/` folder, and `shared/services`, `shared/models`, `shared/interfaces`
- **Exception:** `shared/helpers` has **no barrel** — helpers are imported by full path (`@shared/helpers/number`)

### TypeScript
- `import type` is **required** for all type-only imports (`verbatimModuleSyntax`)
- Component and hook props are a **local `type Props = {}`** per file, not exported; a component that wraps a native element extends that element's HTML attributes type (e.g. `InputHTMLAttributes<HTMLInputElement> & {}`)
- Fixed value sets from the API are modeled as `enum`; entity interfaces are prefixed with `I`
- `as const` is applied to fixed literal arrays/objects (e.g. navigation item lists)

### Path Aliases
- `@components` → `src/components/index.ts` (barrel)
- `@services` → `src/shared/services/index.ts` (barrel)
- `@shared/*` → `src/shared/*` (used for models, interfaces, and helpers)

### Styling
- Tailwind utility classes via `className` are the only styling mechanism
- Dark theme using the **default Tailwind palette** — no custom color tokens. Accent = `amber-*`; surfaces = `zinc-*` / `neutral-*` / `white/N` overlays; danger = `red-*`; status = `green-*` / `orange-*`
- Conditional classes via template literals; `twMerge` when a component merges base classes with a caller-provided `className`
- Animations via `motion/react`

### Language
- **UI strings** are written in **Brazilian Portuguese**
- **Code** (identifiers, types, comments) is written in **English**
